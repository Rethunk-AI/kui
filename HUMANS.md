# KUI — Operator Guide

Setup, configuration, and operations for administrators running KUI. For developer onboarding, see [AGENTS.md](AGENTS.md). For detailed deployment and TLS configuration, see [docs/deployment.md](docs/deployment.md). For the full admin reference, see [docs/admin-guide.md](docs/admin-guide.md).

---

## Prerequisites

| Requirement | Notes |
|-------------|-------|
| **OS** | Linux with KVM/libvirt |
| **libvirt** | `libvirt` and `libvirt-dev` packages installed; `libvirtd` running |
| **Remote hosts** | `nc` (netcat) on any host KUI accesses via `qemu+ssh://`; SSH key auth configured |
| **Go** | 1.22+ with CGO (required for libvirt bindings) |
| **Node / Yarn** | Node.js + Corepack for building the web frontend |
| **Storage** | `/var/lib/kui` writable (SQLite, git data, audit logs); `/etc/kui` for config |

---

## Installation

### Build from source

```bash
git clone https://github.com/Rethunk-AI/kui.git
cd kui
corepack enable          # one-time: activates Yarn 4 via Corepack
make all                 # builds frontend, backend, runs tests and vet
```

The binary is written to `bin/kui`.

### Without libvirt (CI / no KVM)

```bash
make build BUILD_TAGS=
make test BUILD_TAGS=
```

## First-Run Setup

KUI detects a missing config file and enters **setup mode** automatically. No authentication is required until setup completes.

1. Start KUI: `./bin/kui --config /etc/kui/config.yaml --listen :8080`
2. Open the UI in a browser — you are redirected to the setup wizard.
3. Configure:
   - **Admin account** — username and password (stored in SQLite).
   - **Libvirt hosts** — add at least one host. For each host:
     - `id` — short identifier (e.g. `local`, `prod`).
     - `uri` — `qemu:///system` for local, or `qemu+ssh://user@host/system?keyfile=/path/to/key` for remote.
     - `keyfile` — SSH private key path for remote hosts; `null` for local.
   - **Default host** — used for VM operations when none is specified.
4. Complete setup. KUI writes `config.yaml` to disk.
5. Restart KUI (`systemctl restart kui`) to load the new config, then log in.

Setup is idempotent: once a config file exists, the wizard is unavailable. Remove or rename the config file to re-run setup.

## Configuration Reference

Config is written by the setup wizard and is YAML. Location: `/etc/kui/config.yaml` (default) or override with `--config` / `KUI_CONFIG`.

**Required fields:**

| Field | Description |
|-------|-------------|
| `hosts` | List of libvirt connection targets (see setup above) |
| `jwt_secret` | Minimum 32 bytes; generated by setup wizard |

**Common optional fields:**

| Field | Default | Description |
|-------|---------|-------------|
| `default_host` | first host | Host ID used when no host is selected |
| `vm_defaults.vcpus` | 2 | Default vCPUs for new VMs |
| `vm_defaults.memory_mb` | 2048 | Default RAM in MB |
| `vm_defaults.network` | `default` | Default libvirt network |
| `vm_lifecycle.graceful_stop_timeout` | 30s | Wait before force-stopping a VM |
| `db.path` | `/var/lib/kui/kui.db` | SQLite database file |

## Running KUI

### CLI flags

| Flag | Env | Default | Description |
|------|-----|---------|-------------|
| `--config` | `KUI_CONFIG` | `/etc/kui/config.yaml` | Config file path |
| `--listen` | `KUI_LISTEN` | `:8080` | HTTP listen address |
| `--prefix` | `KUI_PREFIX` | *(empty)* | Runtime prefix (re-roots all local paths) |
| `--tls-cert` | — | *(none)* | TLS certificate PEM (requires `--tls-key`) |
| `--tls-key` | — | *(none)* | TLS private key PEM (requires `--tls-cert`) |

### Direct TLS

```bash
./bin/kui --config /etc/kui/config.yaml \
    --listen :8443 \
    --tls-cert /etc/kui/tls/cert.pem \
    --tls-key /etc/kui/tls/key.pem
```

Both `--tls-cert` and `--tls-key` must be supplied together.

### Reverse proxy (production)

Run KUI on localhost and terminate TLS at nginx or Caddy. The proxy must forward WebSocket upgrades (for VNC/serial consoles) and preserve SSE connections (for real-time VM events). See [docs/deployment.md](docs/deployment.md) for nginx and Caddy examples.

## systemd Deployment

A ready-made unit file is provided in `deploy/systemd/`. See [deploy/systemd/README.md](deploy/systemd/README.md) for install instructions, runtime directory setup, and the `kui` system user.

Quick reference:

```bash
sudo cp deploy/systemd/kui.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now kui
sudo journalctl -u kui -f
```

## Contained / Non-Root Mode (`--prefix`)

Use `--prefix` (or `KUI_PREFIX`) to re-root all local file paths under a directory without a real `chroot(2)`. Useful for non-root installs or relocatable trees.

```bash
PREFIX="$HOME/kui-run"
mkdir -p "$PREFIX/etc/kui" "$PREFIX/var/lib/kui"
./bin/kui --prefix "$PREFIX" --config /etc/kui/config.yaml --listen :8080
```

With prefix set, `config.yaml` is read from `$PREFIX/etc/kui/config.yaml`. Libvirt URIs and SSH key paths for remote hosts are **not** re-rooted. See [docs/admin-guide.md](docs/admin-guide.md#contained-non-root-mode-prefix) for full details.

## Common Operations

### Starting and stopping VMs

Use the browser UI. From the VM list, click the action menu for a VM:

- **Start** — powers on the VM.
- **Stop** — graceful shutdown (ACPI), then force-stop after the configured timeout.
- **Force Stop** — immediate power off.
- **Pause / Resume** — suspend and continue execution.
- **Delete** — removes the VM and optionally its disk.

### Console access

- **VNC** — graphical console via noVNC in the browser.
- **Serial** — text console via xterm.js in the browser.

Both require WebSocket support in any reverse proxy.

### Host provisioning

If a libvirt host has no storage pools or networks, KUI can create them. From the setup wizard or the host configuration page, use the **Provision** flow: review the proposed configuration, then confirm. Local hosts only in the current release; remote host provisioning is planned.

## Troubleshooting

| Issue | Check |
|-------|-------|
| Setup wizard not shown | Config file already exists. Remove or rename `/etc/kui/config.yaml` to re-run. |
| Permission denied on provision | KUI must create `/var/lib/kui/images`. Run as root or grant write access to the `kui` user. |
| Host unreachable | Verify `libvirtd` is running on the remote host; SSH key is in `authorized_keys`; `nc` is installed. |
| VNC/serial console fails | Local hosts only in MVP. Remote access requires KUI on the same host as libvirt or a tunnel. See [docs/deployment.md](docs/deployment.md). |
| WebSocket or SSE not working | The reverse proxy must forward `Upgrade` and `Connection` headers and disable buffering for SSE. See [docs/deployment.md](docs/deployment.md). |
| Login loop / 401 | JWT secret mismatch after config change. Restart KUI to reload the secret. |
